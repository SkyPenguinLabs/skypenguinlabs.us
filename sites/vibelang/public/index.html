Not a normal scripting language

Built as a runtime research surface.

V.I.B.E. is currently a lab-based R&D experiment. It is already used in field operations and production applications, but the language is still intentionally experimental, not fully stable, and designed as a proving ground for modern runtime ideas.

Runtime shape

Source to AST to bytecode to VM.

The local system is organized around a clear execution model: source text becomes lexer output, parser output, canonical AST, compiled bytecode, and runtime values inside a VM. Around that path live syntax profiles, execution profiles, modules, plugins, native hooks, audit cases, fuzz harnesses, examples, benchmarks, and a workbench.

Advanced surfaces

Code examples that are supposed to look dangerous.

binary + diagnostics

Packed payloads with inspectable failure state.

V.I.B.E. treats byte work, errors, and evidence as first-class operating material instead of console leftovers.

register("std/data/bin") as bin
register("std/data/bytes") as bytes
register("std/math/hash") as hash
register("std/runtime/errors") as errors

let fault = errors.with_detail(
  errors.wrap(errors.new("demo.failure", "showcase failure"), "wrapping demo"),
  "section",
  "bytecode-stage")

let packet = pack("VnCxa4", 3735928559u, 4660u, 65u, "MOD!")
let widened = bytes.concat(packet, pack("V", 305419896u), "\x90\x90")
print(bin.hexdump(packet), hash.sha256(widened), errors.inspect(fault))
semantic units + algebra

Quant work without unit amnesia.

Semantic unit expressions and algebra modules sit beside runtime source, not in a separate notebook layer.

register("std/math/algebra") as algebra

let display = unit_context("display", 0.0002645833333333333)
unit stride = 3 * ft

let mixed = 12 in + 4 cm
let binary = 1 GiB + 512 MiB
let torque = 1 J as Nm
let restored = unit_deserialize(unit_serialize(96 px))

let svd = algebra.svd([[3, 0], [0, 4]])
let solved = algebra.solve([[2, 1], [1, 1]], [5, 3])
print(unit_format(mixed, "cm"), unit_format(binary, "MiB"), unit_format(torque), unit_format(restored, "px", display), svd.s[0], solved[0])
scheduler + channels

Runtime orchestration as source.

Scheduled functions, channel values, cancellation, and task status become readable source-level flow.

let out = open_channel(8)

let periodic = schedule_every(200, fn() {
  send_value(out, "beat")
  return "beat"
})

pause_func(periodic)
schedule_after(60, fn() {
  resume_func(periodic)
  return "resumed"
})

let first = recv_value(out, 1000)
cancel_func(periodic)
print("CONCURRENCY_ORCHESTRATION_OK", first, task_status(periodic))
one-liners

Perl-shaped pressure, VM-shaped evidence.

The public CLI uses explicit commands, including one-liner execution under `vibe run` for fast operator transforms.

$ vibe run -ane 'print(fields[0], fields[1])' -F: input.txt
left right
alpha beta

$ vibe run -M std/data/text --begin 'print("BEGIN")' \
  -e 'print(line)' --end 'print("END")' -- file-arg

Explicit command suite

`vibe run`, `vibe repl`, `vibe bench`, `vibe config`, `vibe diagnose`, `vibe tool`, and `vibe help` form the public boundary.

Artifact tools

IDX, log, and blob utilities are grouped under `vibe tool`, keeping evidence operations separate from source execution.

Workbench REPL

The interactive workbench exposes staged source, compact environment status, and command-local reference output.

Stdlib modules

Audit corpus

Benchmark lanes

Benchmark commands support suite runs, shell views, focused debug cases, JSON, Markdown, and text reports.
